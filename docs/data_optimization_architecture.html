1. SQL 최적화 (Token Saving)

수만 건의 로우(Row)를 모두 가져오지 않고, DB 레벨에서 1차 가공하여 LLM에게 전달되는 데이터 양을 95% 이상 줄입니다.

  • Dynamic Filtering: 질문에 관련된 지점/날짜만 조회.
  • In-DB Aggregation: Python이 아닌 DB가 합계(SUM)를 계산하여 단일 행 반환.

2. RAG 정밀 타격 (Context Precision)

수백 페이지의 매뉴얼 중 질문과 의미적으로 가장 가까운 Top-3 문맥만 추출하여 할루시네이션을 방지합니다.

  • Vector Search: 단순 키워드가 아닌 '의미(Semantic)' 기반 검색.
  • Thresholding: 유사도가 낮은 쓰레기 정보는 Prompt에 포함시키지 않음.
